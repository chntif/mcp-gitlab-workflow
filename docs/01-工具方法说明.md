# 工具方法说明

本文档面向维护者，按“作用/调用时机/参数/返回/API 映射”说明当前全部工具。

## 1. 通用返回结构

所有工具统一返回：

- `ok: boolean`：是否成功
- `tool: string`：工具名
- `data`：业务结果（结构见各工具的 `outputSchema`）
- `error_type`：`input_error | runtime_error | unknown_error`（失败时）
- `message`：错误信息（失败时）

## 2. Workflow 工具

> 已移除：`workflow_start`、`workflow_complete`

### 2.1 `workflow_parse_requirement`

- 作用：从需求文本推断 `work_type`，生成 `issue_title`、`branch_name`
- 调用时机：创建 Issue 前做预解析
- 关键参数：`requirement_text`（必填），`work_type/summary/label/english_slug`（可选覆盖）
- 返回重点：`work_type`、`commit_type`、`summary`、`issue_title`、`branch_name`
- API 映射：无（纯本地逻辑）

### 2.2 `workflow_analyze_and_create_issue`

- 作用：分析需求并创建 Issue（含标签策略、指派策略）
- 调用时机：需求确定后，进入研发链路前
- 关键参数：
- `requirement_text` 必填
- `issue_project_id`（或 `WORKFLOW_ISSUE_PROJECT_ID`）
- `labels/label` 可选；未传时自动建议标签
- `auto_create_labels` 可选；默认 `true`
- `assignee_usernames/assignee_username` 可选；未传默认指派当前 token 用户
- `issue_template` 可选；未传模板时需传 `expected_behavior/given/when/then`
- 返回重点：
- `parsed`：解析结果
- `issue`：`id/iid/web_url`
- API 映射：
- `GET /users`
- `GET /user`
- `GET /projects/:id/labels`
- `POST /projects/:id/labels`
- `POST /projects/:id/issues`

### 2.3 `workflow_review_mr_and_comment`

- 作用：读取 MR 并发布审查评论，可选审批
- 调用时机：MR 已创建，进入 Code Review 阶段
- 关键参数：
- `code_project_id`（或 `WORKFLOW_CODE_PROJECT_ID`）
- `mr_iid` 必填
- `review_comment_body` 或 `review_summary`
- `approve` 可选
- 返回重点：
- `merge_request`：MR 元数据
- `review_note`：评论结果
- `approval`：审批执行结果（可选）
- API 映射：
- `GET /projects/:id/merge_requests/:mr_iid`
- `GET /projects/:id/merge_requests/:mr_iid/changes`
- `GET /projects/:id/merge_requests/:mr_iid/notes`
- `POST /projects/:id/merge_requests/:mr_iid/notes`
- `POST /projects/:id/merge_requests/:mr_iid/approve`

### 2.4 `workflow_local_sync_checkout_branch`

- 作用：本地仓库同步并切换到目标分支
- 调用时机：MR 创建后，或需要本地跟进远程分支时
- 关键参数：
- `branch_name` 必填
- `repo_path`（或 `WORKFLOW_LOCAL_REPO_PATH`）
- `remote_name`（或 `WORKFLOW_LOCAL_REMOTE_NAME`）
- `base_branch` 可选（或 `WORKFLOW_BASE_BRANCH`）
- 返回重点：
- `commands`：实际执行的 git 命令
- `current_branch`、`head_sha`
- API 映射：无（本地 git 命令）

### 2.5 `workflow_issue_to_mr_full`

- 作用：基于已有 Issue 执行完整交付链路  
  读取 Issue → 建分支 → 提交 commit → 创建 MR → MR 审查评论 → 本地切分支 → 评论 Issue → 更新 issue log
- 调用时机：Issue 已存在，需要自动推进到 MR 阶段
- 关键参数：
- `issue_project_id/code_project_id`（可 env 兜底）
- `issue_iid` 必填
- `base_branch/target_branch`（可 env 兜底）
- `commit_actions`、`change_summary`、`test_plan` 必填
- `checkout_local_branch` 默认 `true`
- `update_log` 默认 `true`
- 返回重点：
- `issue`、`branch`、`commit`、`merge_request`
- `review_note`、`issue_comment`
- `local_checkout`（可选）
- `log`
- API 映射：
- `GET /projects/:id/issues/:issue_iid`
- `POST /projects/:id/repository/branches`
- `POST /projects/:id/repository/commits`
- `POST /projects/:id/merge_requests`
- `GET /projects/:id/merge_requests/:mr_iid/changes`
- `POST /projects/:id/merge_requests/:mr_iid/notes`
- `POST /projects/:id/issues/:issue_iid/notes`
- （可选）`POST /projects/:id/merge_requests/:mr_iid/approve`

### 2.6 `workflow_requirement_to_delivery_full`

- 作用：从需求直达交付（先建 Issue，再执行 `workflow_issue_to_mr_full`）
- 调用时机：希望一条调用覆盖 1+3 全流程
- 关键参数：
- `requirement_text` 必填
- `issue_project_id/code_project_id`（可 env 兜底）
- `commit_actions`、`change_summary`、`test_plan` 必填
- 其余与 `workflow_analyze_and_create_issue` + `workflow_issue_to_mr_full` 对齐
- 返回重点：
- `created_issue`
- `parsed`
- `delivery`（完整交付结果）
- API 映射：组合调用上述两个 workflow 的全部 API

### 2.7 `workflow_append_issue_log`

- 作用：向本地 markdown issue log 追加记录
- 调用时机：需要外部流程补录 issue 到本地日志
- 关键参数：`issue_title/issue_iid/issue_web_url/branch_name/summary/status/log_path`
- 返回重点：`updated/path`
- API 映射：无（本地文件写入）

### 2.8 `workflow_add_issue_comment`

- 作用：给 Issue 写评论（直接 body 或模板生成）
- 调用时机：提交后同步实现说明到 Issue
- 关键参数：`issue_project_id`、`issue_iid`；若不传 `body` 需传模板字段
- 返回重点：`note_id/body/web_url`
- API 映射：`POST /projects/:id/issues/:issue_iid/notes`

### 2.9 `workflow_create_merge_request`

- 作用：基于 Issue/分支创建 MR，可选更新 issue log
- 调用时机：分支和代码已准备好，只需要 MR 阶段
- 关键参数：`issue_iid/branch_name/change_summary/test_plan` + 项目与目标分支
- 返回重点：`merge_request`、`log`
- API 映射：
- `GET /projects/:id/issues/:issue_iid`
- `GET /users`
- `POST /projects/:id/merge_requests`

## 3. GitLab 原子工具

### 3.1 用户与标签

#### `gitlab_get_current_user`

- 作用：获取当前 token 对应用户
- 调用时机：默认指派、调试鉴权身份
- 参数：无
- 返回：用户对象（`id/username/name/...`）
- API：`GET /user`

#### `gitlab_list_labels`

- 作用：查询项目标签列表
- 参数：`project_id` 必填；`search/page/per_page/with_counts/include_ancestor_groups` 可选
- 返回：标签数组
- API：`GET /projects/:id/labels`

#### `gitlab_create_label`

- 作用：创建标签
- 参数：`project_id/name/color` 必填；`description/priority` 可选
- 返回：标签对象
- API：`POST /projects/:id/labels`

#### `gitlab_update_label`

- 作用：更新标签
- 参数：`project_id/name` 必填；`new_name/color/description/priority` 至少一个
- 返回：标签对象
- API：`PUT /projects/:id/labels`

#### `gitlab_delete_label`

- 作用：删除标签
- 参数：`project_id/label_name` 必填
- 返回：`deleted/label_name/project_id`
- API：`DELETE /projects/:id/labels/:label_name`

### 3.2 Issue

#### `gitlab_create_issue`

- 作用：创建 Issue
- 参数：`project_id/title` 必填；其余为 GitLab Issue 可选字段
- 返回：Issue 对象
- API：`POST /projects/:id/issues`

#### `gitlab_get_issue`

- 作用：获取单个 Issue
- 参数：`project_id/issue_iid`
- 返回：Issue 对象
- API：`GET /projects/:id/issues/:issue_iid`

#### `gitlab_get_issue_notes`

- 作用：获取 Issue 评论列表
- 参数：`project_id/issue_iid`；`sort/order_by` 可选
- 返回：评论数组
- API：`GET /projects/:id/issues/:issue_iid/notes`

#### `gitlab_add_issue_comment`

- 作用：给 Issue 写评论
- 参数：`project_id/issue_iid/body`
- 返回：评论对象
- API：`POST /projects/:id/issues/:issue_iid/notes`

#### `gitlab_get_issue_images`

- 作用：提取 Issue 描述和评论中的图片引用，可选下载为 base64
- 参数：`project_id/issue_iid`；`include_notes/include_base64/max_images` 可选
- 返回：`image_count/images[]`
- API：
- `GET /projects/:id/issues/:issue_iid`
- `GET /projects/:id/issues/:issue_iid/notes`
- 图片下载（`full_path` 或 URL）

### 3.3 仓库与文件

#### `gitlab_create_branch`

- 作用：创建分支
- 参数：`project_id/branch/ref`
- 返回：分支对象
- API：`POST /projects/:id/repository/branches`

#### `gitlab_get_file`

- 作用：读取仓库文件并返回 `decoded_content`
- 参数：`project_id/file_path`；`ref` 可选
- 返回：文件对象
- API：`GET /projects/:id/repository/files/:file_path`

#### `gitlab_commit_files`

- 作用：一次 commit 执行多文件 action
- 参数：`project_id/branch/commit_message/actions` 必填；`start_branch` 可选
- 返回：commit 对象
- API：`POST /projects/:id/repository/commits`

#### `gitlab_upload_project_file`

- 作用：上传文件到项目 markdown uploads（常用于 issue/mr 图片）
- 参数：`project_id/filename/content_base64` 必填；`content_type` 可选
- 返回：`url/full_path/markdown` 等
- API：`POST /projects/:id/uploads`

### 3.4 Merge Request

#### `gitlab_get_merge_request`

- 作用：获取 MR 详情
- 参数：`project_id/mr_iid`
- 返回：MR 对象
- API：`GET /projects/:id/merge_requests/:mr_iid`

#### `gitlab_get_mr_notes`

- 作用：获取 MR 评论
- 参数：`project_id/mr_iid`；`sort/order_by` 可选
- 返回：评论数组
- API：`GET /projects/:id/merge_requests/:mr_iid/notes`

#### `gitlab_create_merge_request`

- 作用：创建 MR
- 参数：`project_id/source_branch/target_branch/title` 必填，其余可选
- 返回：MR 对象
- API：`POST /projects/:id/merge_requests`

#### `gitlab_create_mr_note`

- 作用：给 MR 写评论
- 参数：`project_id/mr_iid/body`
- 返回：评论对象
- API：`POST /projects/:id/merge_requests/:mr_iid/notes`

#### `gitlab_get_mr_changes`

- 作用：读取 MR 变更文件和 diff 元信息
- 参数：`project_id/mr_iid`
- 返回：changes 对象
- API：`GET /projects/:id/merge_requests/:mr_iid/changes`

#### `gitlab_approve_mr`

- 作用：审批 MR
- 参数：`project_id/mr_iid`；`sha` 可选
- 返回：审批结果对象
- API：`POST /projects/:id/merge_requests/:mr_iid/approve`

#### `gitlab_unapprove_mr`

- 作用：取消审批
- 参数：`project_id/mr_iid`
- 返回：取消审批结果对象
- API：`POST /projects/:id/merge_requests/:mr_iid/unapprove`
