# 环境变量说明

本文档说明当前服务的全部环境变量、优先级与生效行为。

## 1. 优先级规则

同一配置项的优先级：

1. 工具调用参数
2. 环境变量默认值
3. 无值则返回缺参错误

说明：

- `WORKFLOW_LOCK_*` 是强约束，优先级高于业务逻辑执行
- `GITLAB_TOKEN` 与 `GITLAB_API_BASE_URL` 为启动必填

## 2. 必填环境变量

### 2.1 `GITLAB_TOKEN`

- 作用：GitLab 鉴权 token
- 未设置：服务启动失败
- 影响范围：全部 `gitlab_*` 与 `workflow_*` 工具

### 2.2 `GITLAB_API_BASE_URL`

- 作用：GitLab API 根地址（例如 `https://gitlab.com/api/v4`）
- 未设置：服务启动失败
- 影响范围：全部 GitLab HTTP 请求

## 3. 可选默认值环境变量

### 3.1 项目与分支

- `WORKFLOW_ISSUE_PROJECT_ID`
- Issue 相关默认项目 ID
- `WORKFLOW_ISSUE_PROJECT_PATH`
- Issue 项目路径（用于模板和日志）
- `WORKFLOW_CODE_PROJECT_ID`
- 代码/MR 相关默认项目 ID
- `WORKFLOW_CODE_PROJECT_PATH`
- 代码项目路径（用于评论模板和日志）
- `WORKFLOW_BASE_BRANCH`
- 默认基线分支（建分支、本地同步可用）
- `WORKFLOW_TARGET_BRANCH`
- 默认 MR 目标分支

### 3.2 标签、指派与日志

- `WORKFLOW_LABEL`
- 默认标签（工具未传 label 时可作为 fallback）
- `WORKFLOW_ASSIGNEE_USERNAME`
- 默认指派用户名（工具未传 assignee 时使用）
- `WORKFLOW_ISSUE_LOG_PATH`
- 本地 issue log 默认路径

### 3.3 本地仓库同步

- `WORKFLOW_LOCAL_REPO_PATH`
- 本地仓库路径，供 `workflow_local_sync_checkout_branch` 与全链路 workflow 使用
- `WORKFLOW_LOCAL_REMOTE_NAME`
- 本地 git 远程名（通常 `origin`）

## 4. 强约束环境变量（项目锁）

### 4.1 `WORKFLOW_LOCK_ISSUE_PROJECT_ID`

- 作用：锁定 Issue 工具允许的项目 ID
- 生效后：Issue 相关工具传入其他 project_id 将直接报错

典型受影响工具：

- `workflow_analyze_and_create_issue`
- `workflow_issue_to_mr_full`
- `workflow_requirement_to_delivery_full`
- `workflow_append_issue_log`
- `workflow_add_issue_comment`
- `workflow_create_merge_request`
- `gitlab_create_issue`
- `gitlab_get_issue`
- `gitlab_get_issue_notes`
- `gitlab_add_issue_comment`
- `gitlab_get_issue_images`

### 4.2 `WORKFLOW_LOCK_CODE_PROJECT_ID`

- 作用：锁定代码/MR 工具允许的项目 ID
- 生效后：代码/MR 相关工具传入其他 project_id 将直接报错

典型受影响工具：

- `workflow_analyze_and_create_issue`（若传 `code_project_id`）
- `workflow_issue_to_mr_full`
- `workflow_requirement_to_delivery_full`
- `workflow_review_mr_and_comment`
- `workflow_create_merge_request`
- `gitlab_create_branch`
- `gitlab_get_file`
- `gitlab_commit_files`
- `gitlab_get_merge_request`
- `gitlab_get_mr_notes`
- `gitlab_create_merge_request`
- `gitlab_create_mr_note`
- `gitlab_get_mr_changes`
- `gitlab_approve_mr`
- `gitlab_unapprove_mr`

## 5. 配置建议

### 5.1 最小可用

- 必填 2 项：`GITLAB_TOKEN` + `GITLAB_API_BASE_URL`

### 5.2 团队固定仓库

- 增加 `WORKFLOW_ISSUE_PROJECT_ID`、`WORKFLOW_CODE_PROJECT_ID`、`WORKFLOW_BASE_BRANCH`、`WORKFLOW_TARGET_BRANCH`

### 5.3 本地联动场景

- 增加 `WORKFLOW_LOCAL_REPO_PATH`、`WORKFLOW_LOCAL_REMOTE_NAME`

### 5.4 生产防误操作

- 启用 `WORKFLOW_LOCK_ISSUE_PROJECT_ID` + `WORKFLOW_LOCK_CODE_PROJECT_ID`
